# RPG Engine - Prompt Para Figma

Use este prompt para criar uma primeira base visual e prototipo navegavel do RPG Engine em Figma.

## Prompt

Crie um design system e conjunto de telas desktop-first para um app chamado RPG Engine.

O RPG Engine e uma plataforma para mestres e jogadores de RPG de mesa gerenciarem campanhas, personagens, inventario, magias, bestiario, regras, combate, diario de sessao, chat, IA assistente, PDFs importados e banco de dados/compendio.

Direcao visual:

- Dark fantasy elegante.
- Painel de mestre poderoso, limpo e funcional.
- Inspirado em grimorio, mapa de campanha e mesa virtual.
- Evitar medieval caricato, pergaminho amarelo, ornamentacao exagerada, mascotes e fantasia generica.
- Estetica premium, editorial, noturna e arcana.
- Desktop primeiro, mobile depois.

Cores principais:

- Preto carvao: `#0B0A0C`.
- Obsidiana: `#121015`.
- Pergaminho escuro: `#2A221B`.
- Dourado envelhecido: `#B08A45`.
- Vermelho vinho: `#6F1D2B`.
- Roxo arcano: `#5A3E85`.
- Texto principal: `#E7DDCC`.
- Texto secundario: `#A99B88`.
- Linhas e bordas: `#3A3128`.

Tipografia:

- Titulos com fonte serifada editorial/fantasia, como Cinzel, Cormorant Garamond, EB Garamond ou Spectral.
- Textos longos com fonte sans altamente legivel, como Inter, Source Sans 3, IBM Plex Sans ou Noto Sans.
- Usar monoespacada apenas para rolagens, logs, formulas e dados tecnicos.

Estilo de componentes:

- Sidebar fixa com navegacao por areas.
- Topbar contextual com busca global, campanha ativa, status de sessao e perfil.
- Cards funcionais com raio de 6-8px, borda fina e sombras profundas.
- Tabelas densas com filtros.
- Drawers laterais para detalhes rapidos.
- Badges para secreto, revelado, privado, compartilhado, homebrew, oficial, IA e arquivado.
- Estados de foco com dourado ou roxo.
- Texturas muito sutis de couro escuro, mapa antigo e papel envelhecido escuro.

Crie componentes principais:

- App shell desktop.
- Sidebar.
- Topbar.
- Botao primario, secundario, perigoso e icon button.
- Inputs, selects, textarea editorial e busca global.
- Card de campanha.
- Card de personagem.
- Card de entidade.
- Painel de comando.
- Painel de leitura.
- Tabela densa.
- Badge de permissao.
- Modal.
- Drawer lateral.
- Dice roller.
- Initiative tracker.
- AI assistant panel.
- PDF import card.
- Rule reference block.

Crie telas desktop em 1440px de largura:

1. Login
2. Selecao de campanha
3. Dashboard do Mestre
4. Dashboard do Jogador
5. Ficha de personagem
6. Inventario
7. Grimorio/Magias
8. Bestiario
9. Biblioteca de regras
10. Combate/Iniciativa
11. Diario de sessao
12. Chat da mesa
13. Painel de IA assistente
14. Importador de PDFs
15. Banco de dados/Compendio

Para cada tela, representar:

- Objetivo claro.
- Componentes principais.
- Hierarquia visual.
- Acoes principais.
- Estado vazio relevante.
- Diferenca de permissao entre Mestre e Jogador.

Orientacoes por tela:

- Login: fundo atmosferico escuro, marca forte, formulario limpo e central.
- Selecao de campanha: grid de cards, filtros, convites pendentes e acao de criar campanha.
- Dashboard do Mestre: central de comando com preparacao, segredos, sessoes, encontros, jogadores e atalhos.
- Dashboard do Jogador: personagem ativo, proxima sessao, recursos, revelacoes recentes, diario e chat.
- Ficha de personagem: header com identidade, tabs, atributos, recursos, pericias e notas.
- Inventario: tabela de itens, filtros, moedas, carga, item selecionado em painel lateral.
- Grimorio/Magias: magias preparadas, filtros, detalhes de magia, slots/recursos.
- Bestiario: tabela/lista de criaturas, filtros, statblock, acao adicionar ao combate.
- Biblioteca de regras: busca forte, categorias, leitura editorial e regras relacionadas.
- Combate/Iniciativa: tracker de iniciativa fixo, turno atual destacado, log e painel de acoes.
- Diario de sessao: lista de sessoes, editor por blocos, visibilidade publico/secreto/privado.
- Chat da mesa: canais, feed de mensagens, composer, rolagens e pins.
- Painel de IA assistente: seletor de contexto, prompt, resposta estruturada e acoes de aplicar.
- Importador de PDFs: dropzone, lista de arquivos, status, preview e revisao de entidades.
- Banco de dados/Compendio: colecoes, busca, tabela densa, drawer de detalhes e relacionamentos.

Comportamento visual:

- A interface deve parecer uma ferramenta real de trabalho para campanhas longas.
- Nao criar landing page promocional.
- Nao usar hero com layout marketing.
- Priorizar experiencia de app imediatamente utilizavel.
- Manter leitura confortavel em fundos escuros.
- Usar dourado com moderacao como destaque premium.
- Usar roxo para IA, magia e foco arcano.
- Usar vinho para perigo, combate, dano e segredos.
- Mostrar permissões visualmente por icones e badges.

Entregavel esperado:

- Um frame de design system com tokens, tipografia, cores e componentes.
- Frames desktop para todas as telas listadas.
- Pelo menos uma variante mobile simplificada para Dashboard do Jogador, Chat, Ficha e Busca.
- Prototipo basico conectando Login, Selecao de campanha, Dashboard do Mestre, Dashboard do Jogador e Combate.
